Source Code

The Augur project is divided among several Github repositories:

  • Prototype GUI

    • URL: https://github.com/AugurProject/Augur-UI
    • Maintainer: Scott Leonard
    • Graphical user interface for the Augur prototype. Augur-UI has a clean, responsive HTML5 interface, and is intended to be run as a localhost Flask (Python) app alongside the prototype core. All of the core client's functionality is accessible from the interface. For most users, Augur-UI is the easiest way to use the Augur prototype.

  • Prototype Core

    • URL: https://github.com/AugurProject/AugurCore
    • Maintainer: Zack Hess
    • Python implementation of the Augur/Truthcoin core. Contains all the basic networking, mining, cryptography, event/market creation, and consensus algorithms required for decentralized prediction markets. This code is written as an extension to Zack's earlier project Basiccoin. It includes a suite of CLI commands similar to Bitcoin's RPC command suite.

  • Website

    • URL: https://github.com/AugurProject/augur.net
    • Maintainers: Jack Peterson & Scott Leonard
    • Source code for this website. This may be just a boring, static website, but Jack and Scott built it using Django anyway because they both think Python is pretty okay.

  • SideCoin

    • URL: https://github.com/AugurProject/SideCoin
    • Maintainers: Joey Krug & Jack Peterson
    • SideCoin is a modified version of Bitcoin Core. It allows Bitcoin holders to trustlessly be awarded coins from a "snapshot" of the blockchain (e.g., awarded SideCoins in an amount proportional to the number of Bitcoins they hold). SideCoin was created by Joey and Jack as a solution to the "Spin-off" problem.

  • Time-Lapse Cryptography

    • URL: https://github.com/AugurProject/time-lapse-crypto
    • Maintainer: Alan Lu
    • C++ implementation of the time-lapse cryptography (TLC) protocol, originally outlined by Rabin and Thorpe. Augur and TLC are symbiotic: TLC provides an automatic vote-decryption mechanism for Augur; Augur, in turn, provides a financial incentive for users to reliably maintain "time service" nodes.

  • Snapshot / Blockparser

    • URL: https://github.com/joeykrug/blockparser-memoryLean
    • Maintainer: Joey Krug
    • Joey's memory-lean version of znort's C++ blockparser. We used this modified blockparser to take a "snapshot" of the Bitcoin blockchain -- we took all the public addresses in the network with a substantial balance (over about 20 cents in USD) and sorted them from highest to lowest balance.

  • Consensus algorithm

    • URL: https://github.com/AugurProject/pyconsensus
    • Maintainer: Jack Peterson
    • Standalone Python version of the Principal Component Analysis-based consensus algorithm used by Augur. This implementation was a joint effort between Jack and Paul Sztorc. It is based on R code from the Truthcoin repo.

  • Augur whitepaper

    • URL: https://github.com/AugurProject/whitepaper
    • Maintainer: Jack Peterson
    • Implementation details for the production version of Augur, which we are building as an extension to Bitcoin Core. This paper describes the structure and locking/unlocking Scripts of the various transactions needed to implement prediction markets using only Bitcoin's input/output-style transactions.

  • Truthcoin whitepaper

    • URL: https://github.com/psztorc/Truthcoin
    • Maintainer: Paul Sztorc
    • The theoretical foundation of the Augur project. (It inspired all the rest of this stuff!) This paper covers -- invents, in fact -- the Principal Component Analysis-based consensus algorithm used by our software. It also details Augur's game-theoretic underpinning. This repository includes sample code written in R, Python, and C++.

If you have questions about a particular repo -- or just want to pitch in! -- contact the repo's maintainer, create an issue on Github, or just start hacking away and send us a pull request.
